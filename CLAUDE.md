# CLAUDE.md

## Sprint-Planung

Sprint-Planung immer ueber GitHub Issues strukturieren: Epic-Issue als Sprint-Uebersicht mit allen Tasks als einzelne Issues (Labels fuer Prioritaet P0/P1/P2, Aufwand S/M/L, Kategorie). Jedes Task-Issue referenziert das Sprint-Epic und listet Abhaengigkeiten zu anderen Issues. Verwende GitHub-Features wie Labels, Issue-Referenzen (#N) und Checklisten.
